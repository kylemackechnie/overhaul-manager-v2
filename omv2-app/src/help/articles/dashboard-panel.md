# Dashboard

The Dashboard is the landing page for an active project. It shows project health at a glance — key cost metrics, headcount, schedule progress, and forecast snapshots.

## How the tiles work

The Dashboard is built from **tiles** — small focused widgets, each showing one slice of project data. Tiles draw their data live from across the app (costs, timesheets, resources, POs) so the Dashboard always reflects the current state. No refresh needed.

## Customising the layout

You can personalise which tiles appear and where:

- Drag tiles to reorder them
- Some tiles have their own settings (gear icon in the tile header)
- Layout choices are saved per-user and follow you across devices

If a tile shows blank data, the underlying module probably hasn't been set up yet (no resources, no timesheets, no WBS codes). Set those up in their respective panels and the Dashboard fills in automatically.

## Other dashboards

The Dashboard tab is the project overview, but several modules have their own focused dashboards too:

- **Cost Tracking → Dashboard** for cost-specific metrics
- **Personnel → HR Dashboard** for headcount and roster views
- **HSE → Dashboard** for HSE metrics
- **Tooling → Dashboard** for SE Rental tooling status
- **Equipment Hire → Dashboard** for hire commitments

The main Dashboard pulls highlights from all of these. Drill into the module dashboards for the full picture.
